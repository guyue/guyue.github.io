引用类型

引用类型的值（对象）是引用类型的一个实例。在ECMAScript中，引用类型是一种数据结构，用于讲数据和功能组织在一起。引用类型有时候也被称为对象定义，因为他们描述的是一类对象所具有的属性和方法。

Object类型

我们看到的大多数引用类型值都是Object类型的实例，是ECMAScript中使用最多的一个类型。

创建Object

  • 第一种是使用new操作符后跟Object构造函数
  • 另一种方式是使用对象字面量表示法

在通过对象字面量定义对象时，不会调用Object构造函数

访问对象属性

  • 访问对象属性时使用的都是点表示法
  • javascript也可以使用方括号表示法
  • 方括号语法的主要优点是可以通过变量来访问属性
  • person["first name"] 中间有空格是合法的

Array类型

ECMAScript数组的每一项可以保存任何类型的数据

创建array

  • 第一种是使用Array构造函数
  • 另一种方式是使用数组字面量表示法

在通过数组字面量定义对象时，不会调用Array构造函数

  • 由于数组最后一项的索引始终是length-1，因此下一个新项的位置就是length。每当在数组末尾添加一项后，其length属性都会自动更新以反映这一变化。
  • 数组最多可以包含 4294967295个项

检测数组

  • 对于一个网页或者一个全局作用域而言，使用instanceof操作符
  • ECMAScript 5 使用isArray（）方法

转换方法

所有对象都具有toLocaleString()、toString()和valueOf()方法。其中，调用数组的toString()方式会返回有数组中每个的字符形式拼接而成的一个以逗号分隔的字符串，valueOf()返回的还是数组。如果数组中的某一项的值是null或者undefined，那么该值在join()、toLocaleString()、toString()和valueOf()方法返回的结果中以空字符串表示。

栈方法

  • ECMAScript为数组专门提供了push()和pop()方法，以便实现类似栈的行为。
  • push()方法可以接受任意数量的参数，把他们逐个添加到数组的末尾，并返回修改后数组的长度
  • pop()方法则从数组末尾移除最后一项，减少数组的length，然后返回移除的项

队列方法

  • ECMAScript5为数组专门提供了shift()方法，以便实现类似队列的行为。
  • shift()方法能够移除数组中的第一项并返回该项，同时将数组长度减1
  • unshift()方法能够在数组的前端添加任意个项并返回新数组的长度

IE7以下浏览器 unshift()方法总是返回undefined而不是数组的长度

重排序方法

  • 数组中已经存在两个可以直接用来重排序的方法：reversr()和 sort()
  • reverse()方法会反转数组项的顺序
  • sort() 方法按照升序排列数组项，默认先调用每个数组项的toString方法
  • sort()和reverse()方法的返回值是经过排序之后的数组

操作方法

  • concat() 方法用于连接两个或多个数组,该方法不会改变现有的数组,而仅仅会返回被连接数组的一个副本.
  • slice() 方法可从已有的数组中返回选定的元素,即一个新的数组.该方法可传入2个参数（start,end）,返回的新数组包含从 start 到 end （不包括该元素）的元素，如果 end 未被规定，那么返回start 到数组结尾的所有元素， slice()方法的参数如为负数，则用数组长度加上该数来确定相应的位置。
  • splice() 方法向/从数组中添加/删除项目，然后返回被删除的项目。该方法会改变原始数组。splice() 方法可删除从 index 处开始的零个或多个元素，并且用参数列表中声明的一个或多个值来替换那些被删除的元素。 如果从 arrayObject 中删除了元素，则返回的是含有被删除的元素的数组。
  • 请注意：splice() 方法与 slice() 方法的作用是不同的，splice() 方法会直接对数组进行修改。

位置方法

  • ECMAScript5 为数组实例添加了两个位置方法：indexOf()和lastIndexOf(),这两个方法可返回指定的字符串值在字符串中首次出现的位置，或者在没找到的情况下返回-1。
  • indexOf()方法从数组的开头（位置0）开始向后查找
  • lastIndexOf()方法从数组的末尾开始向前查找

请注意：这两个方法在检索并比较第一个参数与数组中的每一项时，使用全等操作符(===)，且区分大小写，IE8及以下不支持这两个方法

迭代方法

  • every():对数组中的每一项运行给定的函数，如果该函数对每一项都返回true，则返回true
  • filter():对数组中的每一项运行给定的函数，返回该函数会返回true的项组成的数组
  • forEach():对数组中的每一项运行给定的函数，没有返回值
  • map():对数组中的每一项运行给定的函数，返回每次函数调用的结果组成的数组
  • some():对数组中的每一项运行给定的函数，如果该函数对任意一项返回true，则返回true
  • IE8及以下不支持

缩小方法

  • ECMAScript5 为数组实例添加了两个缩小方法：reduce()和reduceRight()
  • reduce和reduceRight用于简化或降级（reduce）数组，前者是从开始处进行分析，而后者则从结束处
  • 回调函数callback接受4个参数：前一个值（previous）、当前值（current）、索引（index）和数组（array）本身
  • IE8及以下不支持

Date类型

ECMAScript中的date类型是在早期java中java.util.Date 类基础上构建的。为此Date类型使用自UTC 1970年1月1日午夜（零食）开始经过的毫秒数来保存日期。在使用这种数据存储格式的条件下，Date类型保存的日期能够精确到 1970年1月1日之前或之后的285616年

继承的方法

与其他引用类型一样，Date类型也重写了toLocaleString()、toString()和valueOf()方法；但这些方法返回的值与其他类型中的方法不同。Date类型的toLocaleString()方法会按照浏览器设置的地区想适应的格式返回日期和时间。Date类型的valueOf()方法，则根本不返回字符串而是返回日期的毫秒数。

日期格式化方法

  • toDateString()——以特定于实现的格式显示星期几、月、日和年
  • toTimeString()——以特定于实现的格式显示时、分、秒和时区
  • toLocaleDateString()——以特定于地区的格式显示星期几、月、日和年
  • toLocaleTimeString()——以特定于现实的格式显示时、分、秒
  • toUTCString()——以特定于现实的格式完整的UTC日期

RegExp类型

ECMAScript通过RegExp类型来支持正则表达式

  • 当您检索某个文本时，可以使用一种模式来描述要检索的内容。RegExp 就是这种模式。
  • 简单的模式可以是一个单独的字符。
  • 更复杂的模式包括了更多的字符，并可用于解析、格式检查、替换等等。
  • 您可以规定字符串中的检索位置，以及要检索的字符类型，等等。

                                    var expression = / pattern/ flags

正则表达式的匹配模式支持以下3个标志。

  • g:表示全局（global）模式
  • i:表示不区分大小写模式
  • m:表示多行模式

RegExp实例属性

  • global：布尔值，表示是否设置了g标志
  • ignoreCase：布尔值，表示是否设置了i标志
  • lastIndex：整数，表示开始搜索下一匹配项的字符位置，从0算起
  • multiline：布尔值，表示是否设置了m标志
  • source：正则表达式的字符串表示，按照字面量形式而非传入构造函数中的字符串模式返回

RegExp实例方法

  • RegExp对象的主要方法是exec(),该方法检索字符串中的指定值。返回值是包含第一个匹配项信息的数组。如果没有发现匹配，则返回 null。
  • test() 方法检索字符串中的指定值。返回值是 true 或 false。
  • RegExp 实例继承的 toLocaleString()和 toString()方法都会返回正则表达式的字面量，与创建正则表达式的方式无关。
  • 正则表达式的 valueOf()方法返回正则表达式本身。

Function类型

没有重载（深入理解）

将函数名想象为指针，有助于了解为什么ECMAScript中没有函数重载的概念。来看下面的例子：


  function addNum(s1){
    return 100+s1;
  }
  function addNum(s2){
    return 200+s2;
  }
  alert(addNum(1)); // 输出 201，实际上第二个函数覆盖了第一个函数。

对于变量的声明，实际上此处的 addNum 是对 Function 对象的一个引用，可以想象成指针，变量声明了两次，实际上第二次的声明切断了函数第一次的引用而指向了另外一个对象。

在这里，我们可以把每个函数都看做是 Function 类型的实例。由于这里的函数是对象，则函数名实际上是指向函数对象的指针，而并不会与某个具体的函数绑定。

函数声明与函数表达式

  • 解析器在向执行环境中加载数据时，对函数声明和函数表达式并非一视同仁。
  • 解析器会率先读取函数声明（函数声明提升），并使其在执行任何代码前可用（可以访问）；
  • 函数表达式，则必须等到解析器执行到它所在的代码行，才会真正被执行

作为值的函数

因为ECMAScript中的函数命名本身就是变量，所以函数也可以作为值来使用。也就是说，不仅可以像传递参数一样把一个函数传递给另一个函数，而且可以将一个函数作为另一个函数的结果返回

函数内部属性

  • 在函数中有两个特殊的对象：arguments 和 this
  • arguments参见第三章，arguments有个callee属性，该属性是一个指针，指向拥有这个arguments对象的函数
  • this引用的是函数据以执行的环境对象，当在网页的全局作用域中调用函数时，this 对象引用的就是 window

函数属性和方法

  • ECMAScript中的函数是对象，包含：length 和 prototype
  • length 属性表示函数希望接收的命名参数的个数
  • prototype 见第六章

Global对象

ECMAScript中的Global对象在某种意义上是作为一个终极的“兜底儿对象”来定义的。换句话说，不属于任何其他对象的属性和方法，最终都是它的属性和方法。事实上，没有全局变量或者全局函数；所有在全局作用域中定义的属性和方法，最终都是他的属性和方法

  • URI编码方法 Global对象的encodeURI()和encodeURIComponent()方法截图对于URI进行编码以便发给浏览器
  • eval()方法 eval方法就像是一个完整的ECMAScript解析器，它只接受一个参数，即要执行的ECMAScript 字符串
  • window对象 详见第八章
